## Methodology

### Scanning Approach

1. **Passive Discovery (ARP)**
   - Queries local ARP table
   - Least intrusive method
   - Discovers hosts on local network segment

2. **Active Discovery (Nmap)**
   - Safe ping scan: `nmap -sn -T2`
   - Polite timing (T2) to minimize network impact
   - No port scanning during discovery phase

3. **Port Scanning**
   - **Limited to:** Ports 22 (SSH), 80 (HTTP), 443 (HTTPS)
   - Safe, non-aggressive scanning
   - No vulnerability scripts
   - No full port scans

### Safety Measures

- ✅ Passive discovery attempted first
- ✅ Safe Nmap flags only (-sn -T2)
- ✅ Limited port scanning scope
- ✅ No aggressive timing
- ✅ No vulnerability exploitation
- ✅ No service version detection scripts

   - Document all discovered hosts and their purposes
   - Track changes in network topology

2. **Service Exposure**
   - Review exposed services on each host
   - Ensure all services are necessary and properly secured
   - Implement principle of least privilege

3. **Network Segmentation**
   - Consider network segmentation for critical assets
   - Isolate sensitive systems from general network traffic
   - Implement firewall rules based on discovered services

4. **Monitoring**
   - Implement network monitoring for unauthorized devices
   - Set up alerts for new devices joining the network
   - Regular scanning to detect changes

5. **Documentation**
